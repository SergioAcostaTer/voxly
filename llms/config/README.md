# 🔧 Configuration & Guidelines

Static reference files for development standards and setup.

## Files

### ai-instructions.md
Development guidelines, coding standards, and best practices for the VoxLy project.
- Code organization patterns
- Naming conventions
- Error handling standards
- Testing requirements
- Documentation standards

### DEVELOPER_CHECKLIST.md
Daily checklist for developers working on VoxLy.
- Morning check-in tasks
- Development workflow
- How to update documentation
- Pull request format
- Testing checklist

## How to Use

1. **Starting a feature?** 
   - Read `ai-instructions.md` first
   - Follow the standards throughout development

2. **During development?**
   - Use `DEVELOPER_CHECKLIST.md` to stay on track
   - Reference `ai-instructions.md` for patterns

3. **Completing a feature?**
   - Follow checklist in `DEVELOPER_CHECKLIST.md`
   - Update documentation in `../planning/backlog.md`
