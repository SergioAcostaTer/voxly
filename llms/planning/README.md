# 📋 Planning & Roadmap

Strategic planning documents for feature development and project timeline.

## Files

### STATUS_CURRENT.md (Main Dashboard)
**Current project status and completion metrics**
- Live status of features
- MVP completion checklist
- Technical stack summary
- Known issues
- Next steps

**Start here for:** Quick overview of what's done and what's next

### backlog.md
Detailed feature list organized by phase and user story.
- 43 total features/tasks
- 145+ story points
- Status tracking (TODO, IN_PROGRESS, DONE)
- File references for each feature
- Phase breakdown

**Start here for:** Picking a task to work on

### backlog.json
Machine-readable version of backlog for LLM processing.
- Structured JSON format
- Easy parsing by tools
- Same information as backlog.md

**Start here for:** Automated tools, task assignment

### roadmap.md
9-phase development timeline from MVP to full release.
- Detailed phase breakdown
- Deliverables per phase
- Duration estimates
- Completion criteria

**Start here for:** Understanding project phases and timeline

## How to Use

### For Project Managers
1. Check `STATUS_CURRENT.md` for quick status update
2. Review `roadmap.md` for timeline
3. Monitor `backlog.md` for progress

### For Developers
1. Read `STATUS_CURRENT.md` to understand where we are
2. Pick task from `backlog.md`
3. Check `roadmap.md` for phase context
4. Reference `../config/ai-instructions.md` for standards

### For Stakeholders
1. Check `STATUS_CURRENT.md` first
2. Ask questions based on `roadmap.md`
3. Review feature list in `backlog.md`
